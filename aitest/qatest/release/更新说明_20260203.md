# 更新说明（2026-02-03）

本次更新主要覆盖：用例创建体验优化、AI 自动化执行在“保存无响应”场景下的自动登记缺陷与自动结束、以及若干稳定性增强。

## 1. 变更摘要

### 1.1 用例管理

- 创建用例页面：默认步骤行会自动预填序号（1..N）。
- 后端保存时：对未删除步骤自动重新归一化序号，保证连续且不缺失。

### 1.2 AI 自动化执行（保存/提交类动作）

- 保存/提交连续两次无任何提示（成功/失败/校验错误等）时：自动创建缺陷并停止执行（避免无限等待/反复点击）。
- 最终步骤断言通过：自动结束执行（避免符合预期但仍继续跑下一轮）。
- submit-like 识别增强：兼容更多 action_script 表达，减少漏判。

## 2. 升级方式（推荐）

适用于“老客户用新包覆盖旧包”。

1) 停止旧版本服务（Web 与执行端）。
2) 用新包代码覆盖旧目录（建议先完整备份旧目录）。
3) 在项目根目录执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\release\upgrade.ps1
```

脚本包含：

- 备份 `db.sqlite3` 与 `.env`（如存在）
- 安装/更新依赖
- 执行数据库迁移
- 收集静态文件

## 3. 启动/重启注意事项

升级后务必同时重启：

- Web 服务：`python manage.py runserver 0.0.0.0:8003`
- AI 执行端（队列 worker）：`python manage.py autotest_worker`

如果只重启 Web、不重启执行端，会出现“页面更新了但执行逻辑仍旧未生效”的错觉。

## 4. 回滚建议

如升级异常：

- 先用 `backup\` 下的 `db.sqlite3`、`.env` 回滚
- 再恢复旧版本代码目录
