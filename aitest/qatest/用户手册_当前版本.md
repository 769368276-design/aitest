# QA Platform 用户手册（当前版本）

本手册重点说明四个高频功能：
- 用例生成
- 用例扩写
- AI测试
- 个人中心 API Key 配置

---

## 1. 开始前准备

1) 先登录系统账号。  
2) 在“项目管理”中确认已有项目。  
3) 在“个人中心”完成模型配置（至少配置“用例生成”和“AI执行”的 Key）。  

---

## 2. 个人中心 API Key 配置

入口：右上角用户名 -> **个人中心** -> **模型配置**

系统分为三个配置页签：
- **用例生成**：用于 AI 生成与扩写
- **AI执行**：用于 AI 测试执行
- **OCR**：用于截图文字识别（可选，但建议配置）

每个页签都需要关注四项：
- Provider（服务商）
- Model（模型名）
- Base URL（接口地址）
- API Key（密钥）

注意事项：
- Key 不回显，留空表示不修改已有 Key。  
- 配置后点击“保存模型配置”。  
- 若未配置对应 Key，生成/扩写/执行会直接报错。  

---

## 3. 用例生成

入口：顶部菜单 **AI生成**

### 操作步骤
1) 选择“所属项目”（可选关联需求）。  
2) 上传文件（支持图片、PDF、OpenAPI JSON/YAML、文本）。  
3) 可补充“需求描述”和“上下文信息”。  
4) 点击“开始生成”。  
5) 生成完成后在右侧查看结果，可：
- 一键导入用例库
- 导出 Excel / XMind
- 复制或清空

### 使用建议
- 输入信息越具体，生成质量越高。  
- 对于接口类需求，优先上传 OpenAPI 文件。  
- 导入前先快速检查标题、步骤、预期结果是否合理。  

---

## 4. 用例扩写

入口：顶部菜单 **用例管理**，在目标用例行点击魔法棒“扩写”按钮。

### 操作步骤
1) 进入“用例管理”列表。  
2) 找到基础用例，点击“扩写”。  
3) 系统弹出“用例扩写”窗口并生成扩写内容。  
4) 你可以在文本框内人工调整。  
5) 点击“确认保存”，系统会把扩写结果解析并入库为新用例。  

### 使用建议
- 基础用例步骤越完整，扩写结果越稳定。  
- 保存前检查是否存在重复场景。  
- 若提示扩写失败，先检查个人中心“用例生成”模型配置。  

---

## 5. AI测试

入口：顶部菜单 **AI测试**

可通过两种方式发起：
- 在“AI测试”页面直接输入标题和步骤执行
- 在“用例管理”列表点击某条用例的“AI执行”按钮

### 执行模式
- 有头模式：弹出浏览器窗口，适合 SSO/内网/证书场景
- 无头模式：后台执行，速度更高

### 运行流程
1) 发起执行后进入控制台页。  
2) 控制台可查看步骤、截图、状态。  
3) 可进行暂停、继续、停止。  
4) 执行结束后点击“查看报告”。  

---

## 6. 测试报告查看

入口：顶部菜单 **测试报告**

报告页可查看：
- 执行状态（Running/Completed/Failed 等）
- 执行轮次（高级用例多轮执行）
- 执行人、开始/结束时间
- 详情与控制台入口

建议重点关注：
- 失败步骤的错误信息
- 关键截图与网络记录
- 结果汇总中的断言信息

---

## 7. 常见问题（简版）

### Q1：点击生成/扩写/执行后提示 Key 相关错误
请到“个人中心 -> 模型配置”补全对应模块的 API Key，并保存。

### Q2：AI测试长时间无进展
优先切换为“有头模式”重试；并检查目标网站登录态、网络连通性与权限。

### Q3：扩写结果保存失败
通常是文本结构不完整。请保留清晰的用例标题与步骤结构后再保存。
