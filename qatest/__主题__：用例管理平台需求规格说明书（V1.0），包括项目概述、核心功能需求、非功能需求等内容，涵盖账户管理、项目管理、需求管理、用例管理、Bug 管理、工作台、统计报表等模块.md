# 用例管理平台需求规格说明书（V1.0）

## 一、项目概述

### 1.1 平台目标

构建一款 **轻量化、流程闭环的用例管理平台**，覆盖 “项目 - 需求 - 用例 - bug” 全测试流程，支持多角色协同，实现需求跟踪、用例标准化管理、bug 全生命周期监控，降低团队沟通成本，提升测试与开发协作效率。

### 1.2 核心用户角色



| 角色    | 核心职责                       |
| ----- | -------------------------- |
| 系统管理员 | 账户管理、角色权限配置、平台参数设置         |
| 项目管理员 | 项目创建、成员分配、流程模板配置           |
| 测试人员  | 需求评审、用例编写 / 执行、bug 提交 / 跟踪 |
| 开发人员  | 需求查看、bug 接收 / 修复 / 反馈      |
| 普通用户  | 查看项目进度、需求文档、用例执行结果（只读权限）   |

### 1.3 适用场景



* 小型 / 中型团队的软件测试流程管理

* 多项目并行的测试任务分配与跟踪

* 需求与用例、用例与 bug 的关联追溯

* 测试数据（用例、bug）的标准化存储与导出

## 二、核心功能需求

### 2.1 账户管理模块（P0 优先级）

#### 2.1.1 注册登录



* 支持手机号 + 验证码注册（需系统管理员审核激活）

* 支持账号密码登录、忘记密码（手机号验证码重置）

* 登录后默认跳转至个人工作台

#### 2.1.2 权限管理



* 系统管理员可创建自定义角色（如 “测试负责人”“开发组长”）

* 支持按角色分配权限（例：仅测试人员可提交 bug，仅项目管理员可删除项目）

* 预定义 3 类基础角色：系统管理员、项目管理员、普通用户（权限不可修改）

#### 2.1.3 个人中心



* 支持修改昵称、密码、手机号、邮箱等个人信息

* 查看个人参与的项目、待办任务（如未处理的 bug、待评审的用例）

* 支持账号注销（需输入密码验证）

### 2.2 项目管理模块（P0 优先级）

#### 2.2.1 项目创建与配置



* 项目管理员可创建项目，填写项目名称、描述、起止时间、负责人

* 支持设置项目状态（待启动 / 进行中 / 已暂停 / 已结束）

* 可上传项目附件（如需求文档、测试计划）

#### 2.2.2 项目成员管理



* 项目管理员可添加 / 移除项目成员，分配成员角色（按项目内权限划分）

* 支持批量导入成员（Excel 模板导入，含账号、角色字段）

* 成员可查看项目内自己的权限范围

#### 2.2.3 项目检索与筛选



* 支持按项目名称、状态、负责人、创建时间搜索筛选

* 个人工作台展示 “我参与的项目”“我负责的项目” 快捷入口

### 2.3 需求管理模块（P0 优先级）

#### 2.3.1 需求创建与编辑



* 关联指定项目，填写需求名称、描述、优先级（高 / 中 / 低）、预计完成时间

* 支持需求分类（如功能需求、非功能需求、优化需求）

* 可上传需求附件（原型图、PRD 文档）

* 支持富文本编辑（插入图片、表格、链接）

#### 2.3.2 需求状态流转



* 预设状态：待评审→已确认→开发中→已完成→已关闭

* 支持自定义状态（项目管理员可添加 / 删除状态，配置流转规则）

* 状态变更时可添加备注（如 “需求评审通过，进入开发”）

#### 2.3.3 需求关联与追溯



* 支持需求与用例关联（一个需求可关联多个用例，一个用例仅关联一个需求）

* 查看需求关联的用例执行情况、关联的 bug 数量（快捷统计）

* 支持需求版本管理（修改需求后保留历史版本，可回滚查看）

### 2.4 用例管理模块（P0 优先级）

#### 2.4.1 用例创建与编辑



* 关联指定需求，填写用例名称、前置条件、操作步骤、预期结果、优先级（高 / 中 / 低）、用例类型（功能用例 / 性能用例 / 兼容性用例）

* 支持用例参数化（如输入值、预期结果可设置多组数据）

* 支持批量复制用例（同项目内或跨项目复制）

#### 2.4.2 用例执行与记录



* 测试人员可标记用例执行状态（未执行 / 执行中 / 通过 / 失败 / 阻塞）

* 执行失败时可直接关联创建 bug（自动带入用例信息、项目信息）

* 记录执行时间、执行人、执行备注（如 “阻塞原因：依赖的接口未开发完成”）

#### 2.4.3 用例检索与导出



* 支持按项目、需求、用例名称、优先级、执行状态搜索筛选

* 支持导出用例为 Excel/Word 格式（含用例全部字段，可自定义导出字段）

* 支持用例批量操作（批量修改优先级、批量执行状态更新）

### 2.5 Bug 管理模块（P0 优先级）

#### 2.5.1 Bug 创建与编辑



* 关联指定项目、用例（可选），填写 bug 标题、描述、复现步骤、严重程度（致命 / 严重 / 一般 / 轻微）、优先级（高 / 中 / 低）

* 支持上传截图 / 录屏附件（最多 5 个文件，单个文件≤100MB）

* 选择影响版本、预期修复版本

#### 2.5.2 Bug 状态流转与分配



* 预设状态：新建→已分配→开发中→已修复→待验证→已关闭 / 重开

* 支持自定义状态流转规则（项目管理员配置）

* 可分配 bug 给项目内开发人员，分配后触发通知（系统消息）

#### 2.5.3 Bug 跟踪与统计



* 显示 bug 全生命周期记录（状态变更、操作人、时间、备注）

* 支持按项目、状态、严重程度、负责人、创建时间筛选

* 快捷查看 “我的待处理 bug”“我提交的 bug”“逾期未修复 bug”

### 2.6 工作台模块（P1 优先级）



* 展示个人待办任务（待执行用例、待处理 bug、待评审需求）

* 项目进度概览（参与项目的需求完成率、用例通过率、bug 修复率）

* 最近操作记录（最近创建 / 编辑的需求、用例、bug）

* 快捷入口（快速创建需求、用例、bug）

### 2.7 统计报表模块（P2 优先级）



* 项目维度：需求完成率、用例执行统计、bug 分布统计（按严重程度 / 状态）

* 个人维度：测试人员用例执行数量、开发人员 bug 修复数量

* 支持报表导出（Excel/PDF）、按时间范围筛选（今日 / 本周 / 本月 / 自定义）

## 三、非功能需求

### 3.1 性能需求



* 支持并发用户数：≥50 人（小型团队场景）

* 页面响应时间：≤2 秒（列表查询、表单提交等核心操作）

* 数据存储：支持≥100 个项目、≥10000 条用例、≥5000 条 bug 的存储

### 3.2 兼容性需求



* 浏览器兼容：支持 Chrome（≥80 版本）、Edge（≥80 版本）、Firefox（≥75 版本）

* 设备兼容：支持 PC 端（Windows/MacOS），适配 1366×768 及以上分辨率

### 3.3 安全性需求



* 密码存储：采用 MD5 加密存储，不明文保存

* 权限校验：所有接口均需验证用户权限，禁止越权操作

* 数据备份：支持手动备份数据库（每周至少 1 次备份提醒）

* 防 SQL 注入：输入内容进行过滤，避免恶意注入攻击

### 3.4 易用性需求



* 操作流程：核心流程（需求→用例→bug）步骤≤3 步，减少跳转

* 界面设计：风格统一，按钮位置直观，支持快捷键（如 Ctrl+S 保存）

* 提示信息：操作成功 / 失败、必填项未填等场景需给出明确提示

* 学习成本：新用户上手时间≤1 小时（基于基础计算机操作能力）

### 3.5 可扩展性需求



* 支持功能模块扩展（如后续新增 “测试计划管理”“接口测试管理”）

* 支持自定义字段（如需求表、用例表可添加自定义字段，适配不同团队需求）

* 支持第三方工具集成（预留 API 接口，可集成 Jenkins、JIRA 等工具）

## 四、界面原型建议（核心页面）

### 4.1 登录页



* 左侧：平台 Logo + 平台介绍；右侧：登录表单（账号 / 手机号、密码、验证码、登录按钮、忘记密码链接）

### 4.2 工作台页



* 顶部：导航栏（首页、项目、需求、用例、bug、报表、个人中心）

* 左侧：项目列表（可折叠）；中间：待办任务卡片、进度统计图表

* 右侧：最近操作记录、快捷创建按钮

### 4.3 用例编辑页



* 顶部：关联项目 / 需求选择栏、保存 / 预览 / 提交按钮

* 主体：表单区域（分标签页：基本信息、用例详情、参数化数据、附件）

* 底部：历史版本记录、操作日志

### 4.4 Bug 列表页



* 顶部：搜索栏（多条件筛选）、批量操作按钮、新增 bug 按钮

* 主体：表格（显示 bug 标题、项目、状态、严重程度、负责人、创建时间）

* 右侧：筛选面板（可固定，按状态 / 严重程度 / 优先级快速筛选）

## 六、需求优先级说明



* P0（必须实现）：核心功能模块（账户、项目、需求、用例、bug 管理的基础功能）+ 安全性需求

* P1（建议实现）：工作台模块 + 核心非功能需求（性能、兼容性、易用性）

* P2（后续迭代）：统计报表模块 + 可扩展性需求 + 高级功能（如批量导入、第三方集成）
