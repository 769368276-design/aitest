# 本地安装说明（Windows / macOS）

本包用于**纯本地安装**：在你自己的电脑上启动一个本地 Web 服务，然后用浏览器访问 `http://localhost:8003/`。不涉及服务器部署、不需要 Docker。

---

## 1. 你要做的事（最快路径）

### Windows
1）解压后进入项目根目录（能看到 `manage.py`、`requirements.txt`、`安装点我.bat`）  
2）双击 `安装点我.bat`  
3）看到提示 `Starting server: http://localhost:8003/` 后，用浏览器打开

### macOS
1）解压后进入项目根目录（能看到 `manage.py`、`requirements.txt`、`安装点我.command`）  
2）双击 `安装点我.command`  
3）看到提示 `Starting server: http://localhost:8003/` 后，用浏览器打开

---

## 2. 必要前置（第一次必须）

### 2.1 Python
- 需要 Python **3.12+**
- Windows：安装时勾选 “Add Python to PATH”
- macOS：推荐安装 Python 3.12+（python.org 或 Homebrew 均可）

验证命令（能输出版本号才算成功）：
- Windows（PowerShell）：

```powershell
python --version
```

- macOS（Terminal）：

```bash
python3 --version
```

---

## 2.2 数据库（默认本地 sqlite）

本地安装默认使用 sqlite（数据库文件为项目根目录的 `db.sqlite3`），不需要安装/配置 PostgreSQL。

脚本在第一次运行时会优先复制 `.env.local.example -> .env`（如果不存在才回退用 `.env.example`）。

---

## 3. 离线/企业网络（避免 pip 代理导致装不上）

本包支持“同目录 wheelhouse 离线安装”：
- 如果项目根目录存在 `wheelhouse/`，脚本会优先用它离线安装依赖（不走公网、不受代理影响）
- 如果没有 `wheelhouse/`，脚本会尝试联网安装依赖

### 3.1 pip 被代理/公司策略干扰时怎么办？
默认安装脚本会尽量忽略你电脑上的全局 pip 配置（例如 `pip.ini/pip.conf` 里的代理/源配置），以避免“明明有网但 pip 走错代理导致连不上”。

如你确实需要使用公司代理/私有源，请在运行前设置：
- Windows（PowerShell）：

```powershell
$env:QA_ALLOW_PIP_CONFIG=1
.\scripts\windows_oneclick.ps1
```

- macOS（Terminal）：

```bash
export QA_ALLOW_PIP_CONFIG=1
./scripts/macos_oneclick.sh
```

### 3.2 是否需要升级 pip？
默认脚本**不会主动升级 pip**（避免某些 Windows 环境下升级 pip 触发权限/路径异常）。如你需要升级，请显式开启：
- Windows（PowerShell）：

```powershell
$env:QA_UPGRADE_PIP=1
.\scripts\windows_oneclick.ps1
```

- macOS（Terminal）：

```bash
export QA_UPGRADE_PIP=1
./scripts/macos_oneclick.sh
```

## 4. Playwright 浏览器安装（AI 测试需要）

AI 测试功能需要 Playwright 的 Chromium 浏览器依赖。默认脚本会自动安装并做一次自检。

浏览器会下载到项目目录下的 `.playwright/`（便于拷贝/离线复用）。

如果你只想先把系统跑起来（暂时不做 AI 测试），可以跳过：
- Windows（PowerShell）：

```powershell
$env:QA_SKIP_PLAYWRIGHT=1
.\scripts\windows_oneclick.ps1
```

- macOS（Terminal）：

```bash
export QA_SKIP_PLAYWRIGHT=1
./scripts/macos_oneclick.sh
```

## 5. 常用参数（不用改代码）

- 换端口（默认 8000）：
  - Windows：

```powershell
$env:QA_PORT=8010
.\scripts\windows_oneclick.ps1
```

  - macOS：

```bash
export QA_PORT=8010
./scripts/macos_oneclick.sh
```

- 只安装不启动：
  - Windows：

```powershell
$env:QA_INSTALL_ONLY=1
.\scripts\windows_oneclick.ps1
```

  - macOS：

```bash
export QA_INSTALL_ONLY=1
./scripts/macos_oneclick.sh
```

- 禁用 wheelhouse（强制联网安装）：
  - Windows：

```powershell
$env:QA_NO_WHEELHOUSE=1
.\scripts\windows_oneclick.ps1
```

  - macOS：

```bash
export QA_NO_WHEELHOUSE=1
./scripts/macos_oneclick.sh
```

## 6. 常见报错排查（最常见三类）

### 6.1 找不到 python
- 现象：提示 `Python not found` / `python not found`
- 处理：安装 Python 3.12+ 并加入 PATH，然后重新运行

### 6.2 依赖安装失败（网络/代理/证书）
- 现象：pip 报连接超时、代理错误、证书错误
- 处理优先级：
  1）使用带 `wheelhouse/` 的离线包（推荐）  
  2）在公司网络下需要代理/私有源时，设置 `QA_ALLOW_PIP_CONFIG=1` 再运行  
  3）换到可直连公网网络再运行

### 6.3 Playwright 浏览器下载失败
- 现象：`python -m playwright install chromium` / `playwright install chromium` 下载失败，或报 `[WinError 2] 系统找不到指定的文件`
- 处理：
  1）确保用项目虚拟环境执行：`.\.venv\Scripts\python.exe -m playwright install chromium`  
  2）如果需要镜像/多源重试：运行 `.\release\download_playwright_browsers.ps1 -Browser chromium -BrowsersPath .\.playwright -PlaywrightDownloadHost https://npmmirror.com/mirrors/playwright`  
  3）先跳过 `QA_SKIP_PLAYWRIGHT=1` 把系统跑起来，再补装  
  4）如果你拿到了包含 `.playwright/` 的离线浏览器目录，把它放到项目根目录即可复用

### 6.4 数据库连接报错：failed to resolve host 'db'
- 现象：运行 `migrate` 或启动时报 `failed to resolve host 'db'`
- 原因：当前 `.env` 使用了 Docker/服务器用的 Postgres 配置（`DJANGO_DB_ENGINE=postgres` 且 `DJANGO_DB_HOST=db`）
- 处理（推荐做法）：
  1）删除项目根目录的 `.env`  
  2）重新运行一键脚本，让它自动用 `.env.local.example` 生成本地 sqlite 配置

### 6.5 目录报错：PathNotFound / cd 失败
- 现象：提示目录不存在、`cd ...` 失败、或运行后找不到 `manage.py`
- 原因：没有在“项目根目录”运行（项目根目录必须能看到 `manage.py`、`requirements.txt`、`安装点我.*`）
- 处理：重新解压到一个普通目录，然后从项目根目录双击启动脚本（不要在子目录里运行）

## 7. 启动入口（已安装依赖后）

### Windows

```powershell
.\.venv\Scripts\python.exe manage.py migrate
.\.venv\Scripts\python.exe manage.py runserver 0.0.0.0:8003
```

### macOS

```bash
./.venv/bin/python manage.py migrate
./.venv/bin/python manage.py runserver 0.0.0.0:8003
```
